# MPI Parallel CFD Solver - MOVERS-N

## Overview

This is a parallelized **2D compressible CFD** solver with **Immersed Boundary Method (IBM)** and **MPI** domain decomposition in the *i*-direction. By default it solves the **laminar Navier–Stokes equations** using the **MOVERS-N** scheme (central inviscid flux + MOVERS-N dissipation + Newtonian viscous terms) with **no-slip, adiabatic IBM walls**. Set `USE_NAVIER_STOKES` to `0` in `SolverOptions.h` for legacy **inviscid Euler + slip** IBM.

## Features

- **MPI Parallelization**: Domain decomposition in the i-direction with ghost cell communication
- **Immersed Boundary Method**: Handle complex geometries without body-fitted grids
- **MOVERS-N Flux Scheme**: Robust upwind flux calculation
- **Optimized Performance**: Compiler optimizations and efficient memory access patterns
- **Scalable Architecture**: Designed for multi-core and cluster computing
- **Navier–Stokes (default)**: Laminar viscosity, viscous flux on fluid and fluid–wall faces, IBM **no-slip** velocity
- **IBM thermal wall**: Adiabatic (default) or fixed `T_w` — see `SolverOptions.h`
- **Optional WENO-5**: Higher-order inviscid interface (`USE_WENO5`)
- **Legacy Euler**: `USE_NAVIER_STOKES 0` → inviscid slip IBM, no viscous flux

## Optional Navier–Stokes and WENO-5

**Default physics** (`SolverOptions.h`): **Navier–Stokes** with laminar viscosity, **IBM no-slip** wall, and **adiabatic** wall temperature. Set `USE_NAVIER_STOKES` to `0` to recover legacy **Euler + slip** IBM only.

**IBM wall thermal** (in `SolverOptions.h`, only when Navier–Stokes is on):

| Setting | Defines | Wall temperature |
|--------|---------|------------------|
| Adiabatic (default) | `IBM_WALL_ADIABATIC 1`, `IBM_WALL_FIXED_TW 0` | \(\partial T/\partial n = 0\) → \(T_\mathrm{ghost} = T_\mathrm{image}\) |
| Fixed \(T_w\) | `IBM_WALL_FIXED_TW 1` | \(T_\mathrm{ghost} = 2 T_w - T_\mathrm{image}\), set `WALL_TEMPERATURE_TW` |

**IBM velocity**: image bilinear + **no-slip** \( \mathbf{u}_\mathrm{ghost} = -\mathbf{u}_\mathrm{image}\). Viscous flux on **fluid–fluid** and **fluid–IBM** faces.

**WENO** (optional, `USE_WENO5=1`): component-wise WENO-5 on \((\rho,u,v,P)\); MOVERS-N dissipation unchanged.

```bash
# Legacy Euler + slip IBM only
make NO_OPENMP=1 CXXFLAGS='-O3 -Wall -std=c++11 -march=native -mtune=native -DUSE_NAVIER_STOKES=0' \
  MPIFLAGS='-O3 -Wall -std=c++11 -march=native -mtune=native -DUSE_NAVIER_STOKES=0'

# Isothermal IBM wall (example)
# Edit SolverOptions.h: IBM_WALL_FIXED_TW 1, WALL_TEMPERATURE_TW 1.0
```

## Files Description

### Core Solver Files
- `MOVERSNEulerSolverMPI.cpp` - MPI parallel version of the CFD solver
- `MOVERSNEulerSolverIBMV4.2.cpp` - Serial version with IBM
- `LocatorV4.1.cpp` - Grid generation and immersed boundary preprocessing

### Header Files
- `MathFunctions.h` - Mathematical utility functions (eigenvalues, flux calculations)
- `LocatorFunctions.h` - Geometric utility functions
- `SolverFunctions.h` - General solver utilities
- `SolverOptions.h` - `USE_NAVIER_STOKES` (default 1), `USE_WENO5`, μ, Pr, IBM adiabatic / fixed `T_w`
- `IbmBoundary.h` - No-slip / wall-function / slip IBM velocity; wall temperature; `viscousFaceActive`
- `Physics.h` - `updateAuxiliary` (primitive → \(E,a,H,T,\mathrm{Mach}\))
- `HighOrderFlux.h` - `eulerFluxDotN` for WENO / inviscid closure
- `WENO5.h` - Jiang–Shu WENO-5 scalar reconstruction
- `ViscousFlux.h` - Newtonian stress, heat flux, viscous CFL helper

### Input Files (Required)
- `GridPoints.dat` - Grid coordinates (generated by `LocatorV4.1`)
- `ImmersedBoundary.dat` - Immersed boundary data (generated by locator)

Geometry definition files (`.txt`) live in the project’s **`../geometry/`** folder. Copy or link them here if the locator expects them in the current directory.

### Output Files
- `Residues_MPI.dat` - Convergence residuals
- `Aerospike_IBM_MPI_Solution_*.dat` - Flow solution at specified time steps

Archived outputs can be moved to **`../output/mpi_solver/`** to keep this directory clean.

## Usage

### Running the MPI Solver
```bash
# Run with 4 processes
make run-mpi

# Run with custom number of processes
make run-mpi-np NP=8

# Direct mpirun command
mpirun -np 4 ./MOVERSNEulerSolverMPI
```

```bash

```


## Input File Formats

### GridPoints.dat
```
Nx Ny
x[0][0] y[0][0]
x[0][1] y[0][1]
...
```

### Wall function (optional)

Compile with `-DUSE_WALL_FUNCTION=1` for a blended laminar / log-law shear model on IBM walls (coarse near-wall grids). Default is resolved no-slip (`USE_WALL_FUNCTION=0`). Low-speed regression case:

```bash
cd cases/viscous_channel && bash run_test.sh
```

### ImmersedBoundary.dat
```
Nx Ny
Cx Cy Fl Gst Rad ImNx ImNy ImX ImY ImBx ImBy Geomx Geomy ImGx ImGy
...
```

Where:
- `Cx, Cy`: Cell center coordinates
- `Fl`: Fluid cell flag (1=fluid, 0=solid)
- `Gst`: Ghost cell flag
- `Rad`: Distance to nearest boundary
- `ImNx, ImNy`: Image point normal vectors
- `ImX, ImY`: Image point coordinates
- `ImBx, ImBy`: Image point indices
- `Geomx, Geomy`: Geometry point coordinates
- `ImGx, ImGy`: Geometry point indices

## MPI Implementation Details

### Domain Decomposition
- 1D decomposition in the i-direction (can be extended to 2D)
- Ghost cell layers for inter-process communication
- Load balancing handles remainder cells

### Communication Pattern
- Non-blocking MPI communication (MPI_Isend/MPI_Irecv)
- Ghost cell exchange every time step
- Global reductions for residuals and time step

### Memory Layout
- All processes store full grid coordinates (broadcast from rank 0)
- Conservative variables distributed across processes
- Communication buffers for ghost cell exchange

## Performance Considerations

### Optimal Process Count
- Best performance typically achieved with process count dividing grid size evenly
- Consider memory per process and communication overhead
- Test with different process counts for your specific case

### Scaling Guidelines
- Good scaling up to ~100-200 processes for typical grid sizes
- Communication becomes dominant for very high process counts
- Consider 2D decomposition for larger process counts

## Troubleshooting

### Common Issues

1. **File Not Found Errors**
   - Ensure `GridPoints.dat` and `ImmersedBoundary.dat` exist
   - Check file format and data consistency

2. **MPI Errors**
   - Verify MPI installation: `mpirun --version`
   - Check process count doesn't exceed available cores
   - Ensure consistent MPI environment across nodes

3. **Convergence Issues**
   - Reduce CFL number if solution diverges
   - Check grid quality and boundary conditions
   - Verify initial conditions are physical

4. **Performance Issues**
   - Profile communication vs computation time
   - Consider different domain decomposition strategies
   - Check memory bandwidth limitations

## Theory and Method

### Governing Equations
2D Euler equations in conservative form:
```
∂U/∂t + ∂F/∂x + ∂G/∂y = 0
```

Where U = [ρ, ρu, ρv, ρE]ᵀ

### Spatial Discretization
- Finite volume method on unstructured grids
- MOVERS-N upwind flux scheme
- Artificial dissipation for stability

### Time Integration
- Explicit time stepping
- Local time stepping for steady-state
- CFL-based time step calculation

### Immersed Boundary Method
- Image point interpolation
- Ghost cell mirroring for wall boundaries
- Fluid/solid interface treatment
